1
Instalando os Pré-Requisitos
Node.js, Git, GitHub CLI, Claude Code — pelo site ou por comando
▼

Ambas 1.1 — Node.js (motor que roda projetos JavaScript — sem ele, nada funciona)

Acesse nodejs.org → clique no botão verde LTS (versão estável) → instale clicando "Next" em tudo.

PowerShell (Admin)
winget install --id OpenJS.NodeJS.LTS --accept-source-agreements

⚠️ Abra o PowerShell como Administrador (botão direito → "Executar como administrador"). Depois feche e reabra.

Verifique se instalou:

PowerShell
node --version
npm --version
✅ Apareceu algo como v20.x.x? Perfeito, está instalado!

Ambas 1.2 — Git (ferramenta que conecta seu computador ao GitHub)

Acesse git-scm.com → instale clicando "Next" em tudo.

PowerShell (Admin)
winget install --id Git.Git --accept-source-agreements

Configure seu nome e e-mail (mesmos do GitHub):

PowerShell
git config --global user.name "Seu Nome Aqui"
git config --global user.email "seuemail@exemplo.com"
📌 O Git é obrigatório também para o Claude Desktop: sem ele, a aba Code não funciona no Windows.

Ambas 1.3 — GitHub CLI (cria Pull Requests pelo terminal)

PowerShell (Admin)
winget install --id GitHub.cli --accept-source-agreements

Ou baixe em cli.github.com. Depois faça login:

PowerShell
gh auth login
# Escolha: GitHub.com → HTTPS → Yes → Login with browser

1.4 — Claude Code

🔑 Duas formas de autenticar (escolha UMA):
• Conta Pro ou Max (assinatura em claude.ai) — mais simples, faz login e pronto
• API Key (paga por uso em console.anthropic.com) — alternativa avançada

⚠️ Se tem conta Pro ou Max, NÃO precisa de API Key.
PowerShell
npm install -g @anthropic-ai/claude-code
# Depois, em qualquer pasta de projeto:
claude
# Na primeira vez, escolha "Claude.ai account" para login com Pro/Max

1. Baixe o app em claude.ai/download
2. Instale e faça login com sua conta Pro ou Max
3. Clique na aba "Code" (ao lado do chat)
4. Em "Environment" → selecione "Local"
5. Selecione a pasta do projeto

NÃO precisa instalar via npm. O app já inclui tudo.

📝 Fase 1 — Briefing (O que você quer construir?)

Descreva o que quer em português. O @pm (Gerente de Produto) transforma isso em um documento técnico.

Exemplo prático
@pm *gather-requirements
# O PM vai te perguntar sobre o projeto. Responda em português.
# Depois ele cria o PRD (Product Requirements Document).

# Ou descreva tudo de uma vez:
Quero criar um sistema de agendamento para clínica odontológica.
Precisa ter: cadastro de pacientes, agenda do dentista,
confirmação por WhatsApp e painel administrativo.

🔍 Fase 2 — Análise + 🏗️ Fase 3 — Arquitetura

O @analyst pesquisa o que é necessário. O @architect projeta como tudo se conecta.

Copie em sequência
@analyst *research-deps       # Pesquisa bibliotecas necessárias
@architect *assess-complexity  # Avalia complexidade
@architect *create-plan        # Cria plano de implementação
@architect *create-context     # Cria contexto do projeto

📋 Fase 4 — Stories + 💻 Fase 5 — Código

O @sm cria histórias detalhadas. O @dev implementa cada uma.

Copie em sequência
@sm *create-story             # Cria história de desenvolvimento
@dev *execute-subtask 1.1      # Implementa subtarefa 1
@dev *execute-subtask 1.2      # Implementa subtarefa 2
# Continue até completar todas as subtarefas

🧪 Fase 6 — Testes + 🚀 Fase 7 — Deploy

O @qa revisa qualidade e segurança. O @devops publica.

Copie em sequência
@qa *review-build STORY-1     # QA revisa o código
@qa *request-fix "Descrição"  # Se achou problema
@dev *apply-qa-fix             # Dev corrige
@qa *verify-fix               # QA confirma
@devops *deploy               # Publica o projeto

📖 Vocabulário de Programação para Leigos

Clique em cada termo para ver a definição, analogia e como pedir ao Claude Code. Baseado no Manual de Vocabulário para Programação.

🧱 Fundamentos do Código
Código / Codelê-se: "code"
▼
As instruções escritas que dizem ao computador o que fazer. É como uma receita de bolo: passo a passo, numa linguagem que a máquina entende.
🍳 Se o programa é um prato pronto, o código é a receita que ensina como preparar.
Repositório (Repo)lê-se: "repo"
▼
A pasta principal do projeto. Guarda todo o código, histórico de alterações e arquivos. Normalmente fica no GitHub.
📁 É o armário de documentos do projeto — tudo fica ali organizado, com registro de quem mexeu no quê e quando.
💬 "Crie um novo repositório para o meu projeto de loja online."
Frontend e Backendlê-se: "front-end" e "bec-end"
▼
Frontend é tudo que o usuário vê e clica — botões, cores, textos, layout. Backend é a parte invisível — o motor que processa dados e faz a lógica funcionar.
🍽️ Frontend é o salão do restaurante (o que o cliente vê). Backend é a cozinha (onde tudo acontece por trás).
💬 "Mude a cor do botão no frontend." / "Faça o backend salvar o e-mail do usuário."
APIlê-se: "ei-pi-ai"
▼
Um canal de comunicação entre sistemas. Permite que um programa converse com outro para trocar dados.
🧑‍🍳 É o garçom do restaurante — leva seu pedido (requisição) para a cozinha (servidor) e traz o prato pronto (resposta).
💬 "Integre com a API do WhatsApp para enviar mensagens automáticas."
Banco de Dados (Database)lê-se: "deita-beis"
▼
Onde o programa guarda as informações de forma organizada: cadastros, produtos, pedidos. Exemplos: PostgreSQL, MySQL, MongoDB.
🗄️ É o fichário do escritório — cada gaveta tem informações organizadas por categoria.
💬 "Crie um banco de dados para armazenar os clientes com nome, email e telefone."
Framework e Bibliotecalê-se: "freim-uorc"
▼
Ferramentas prontas que ajudam a programar mais rápido. Framework é uma estrutura completa (ex: Next.js). Biblioteca é uma ferramenta específica (ex: React).
🏠 Framework é uma casa pré-fabricada. Biblioteca é uma ferramenta da caixa de ferramentas.
🔄 Versionamento (Git)
Commitlê-se: "comit"
▼
Um ponto de salvamento no código. Registra as mudanças com uma mensagem. Você pode voltar a qualquer commit anterior.
💾 É como o Ctrl+S com superpoderes — salva E cria um registro na linha do tempo.
💬 "Faça um commit com a mensagem: 'adiciona página de contato'."
Branchlê-se: "brench" (ramo)
▼
Uma cópia paralela do código onde você trabalha sem bagunçar a versão principal.
📝 Imagine um rascunho separado do documento oficial. Você experimenta à vontade.
💬 "Crie uma branch chamada 'nova-pagina-de-precos'."
Mergelê-se: "merdj" (juntar)
▼
Juntar o código de uma branch com outra. Quando termina, faz merge para a branch principal.
📋 É colar o rascunho no documento oficial.
Pull Request (PR)lê-se: "pul ri-quest"
▼
Um pedido formal para juntar seu código à branch principal. Como dizer: "terminei, revise e aprove".
📄 É entregar um relatório para o chefe revisar antes de publicar.
💬 "Abra um Pull Request desta branch para a main."
Push e Pulllê-se: "puch" e "pul"
▼
Push = enviar suas alterações para o GitHub. Pull = baixar as alterações do GitHub para seu computador.
☁️ Push é fazer upload no Google Drive. Pull é baixar a versão mais recente do Drive.
Clonelê-se: "clone"
▼
Copiar um repositório inteiro do GitHub para o seu computador, incluindo todo o histórico.
📂 É fazer uma cópia completa de um dossiê para trabalhar na sua mesa.
🚀 Execução e Entrega
Deploylê-se: "deploi" (publicar)
▼
Colocar o projeto no ar para que outras pessoas acessem pela internet.
🏪 É a inauguração da loja — o público pode entrar.
💬 "Me ajude a fazer o deploy deste projeto na Vercel."
Buildlê-se: "bild" (construir)
▼
Transformar o código-fonte numa versão otimizada e pronta para funcionar.
🧳 É preparar a mala para a viagem — pegar tudo, organizar e fechar.
Ambientes: Local, Staging, Produçãolê-se: "stei-djing"
▼
Local = seu computador. Staging = cópia de teste. Produção = ao vivo para os usuários.
🎭 Local é a cozinha de casa. Staging é o ensaio geral. Produção é a noite de estreia.
📦 Estrutura do Projeto
Componentelê-se: "com-po-nen-te"
▼
Um pedaço reutilizável da interface (botão, menu, card). Monta a página como Lego.
🧱 Cada peça de Lego é um componente. Você combina para montar o que quiser.
💬 "Crie um componente de card de depoimento com foto, nome e texto."
Rota (Route)lê-se: "raut"
▼
O endereço/caminho de uma página. Ex: /sobre → Sobre Nós, /contato → Contato.
🏷️ Cada rota é uma placa de endereço dentro do site.
💬 "Crie uma rota /dashboard que mostra o painel do usuário."
Variável de Ambiente (.env)lê-se: "dot env"
▼
Informações secretas (senhas, chaves de API) num arquivo separado que nunca é compartilhado.
🔐 É o cofre do escritório — guarda senhas longe de olhos curiosos.
💬 "Coloque a chave da API do Stripe como variável de ambiente."
Dependência (Package)lê-se: "pequedj"
▼
Pacotes de código prontos que você instala para não reinventar a roda. Ficam em package.json.
🛒 É comprar ingredientes semi-prontos em vez de plantar do zero.
🐛 Erros e Problemas
Buglê-se: "bag"
▼
Um erro no código que faz o programa se comportar de forma inesperada.
🧂 Um erro de digitação na receita — "10 kg de sal" em vez de "10 g".
💬 "Tem um bug: quando clico em 'Comprar', nada acontece. Investigue e corrija."
Debuglê-se: "di-bag"
▼
Investigar e corrigir bugs. Examinar pistas (logs, erros) para encontrar a causa.
🔧 É o mecânico abrindo o capô para descobrir o barulho estranho.
Loglê-se: "log" (registro)
▼
Registro automático de tudo que acontece no programa. Consulte quando algo der errado.
📒 É o livro de ocorrências do porteiro.
💬 "Me mostre os logs de erro para entender por que o login não funciona."
Erros HTTP (404, 500...)
▼
404 = página não encontrada. 500 = erro interno do servidor. 401/403 = sem permissão.
🏚️ 404 é chegar a um endereço e descobrir que o prédio não existe. 500 é o prédio existir mas ter um curto-circuito.
Refatorar (Refactor)lê-se: "ri-fac-tor"
▼
Reorganizar o código sem mudar o que ele faz. Deixar mais limpo e fácil de entender.
🗄️ Reorganizar um armário bagunçado — as roupas são as mesmas, mas agora acha tudo mais rápido.
💬 "Refatore o código do login para ficar mais organizado."
🏆 Dica de Ouro: Você não precisa ser técnico. Descreva o que QUER que aconteça ("quando o usuário clicar aqui, quero que apareça uma mensagem de sucesso") e o Claude Code transforma em código.

❓ Perguntas Frequentes

Dúvidas comuns de quem está começando. Clique para expandir.

Preciso saber programar para usar tudo isso? ▼
Não! O Claude Code e os agentes AIOS entendem português natural. Basta descrever o que quer (ex: "crie um botão azul que salva o nome do cliente") e eles transformam em código. O vocabulário deste guia ajuda você a se comunicar melhor, mas não é obrigatório saber programar.
Qual a diferença entre Terminal e Claude Desktop? ▼
Terminal (CLI): Tela preta, digita comandos, mais controle. Precisa instalar Claude Code via npm. Desktop: App visual com aba "Code", clica em botões, vê diffs (diferenças) visualmente, sessões paralelas. Já vem no app. O motor é o mesmo — os agentes e comandos funcionam igual nas duas opções.
Preciso pagar? Quanto custa? ▼
Você precisa de uma conta paga do Claude: Pro (~US$20/mês) ou Max (~US$100/mês). A Pro é suficiente para começar. NÃO precisa de API Key — basta fazer login com sua conta. Lovable e GitHub têm planos gratuitos.
Funciona no Mac/Linux? ▼
Sim! O guia foca no Windows/PowerShell, mas Node.js, Git e Claude Code funcionam em qualquer sistema. No Mac, use o Terminal em vez do PowerShell. Os comandos são praticamente idênticos. No Linux, instale via gerenciador de pacotes (apt, brew, etc.).
O que é "winget" que aparece nos comandos? ▼
winget é o gerenciador de pacotes do Windows — uma loja de apps que funciona por comando. Vem pré-instalado no Windows 11 e em versões atualizadas do Windows 10. Se não funcionar, instale as ferramentas pelo site (as opções estão nas abas do guia).
Deu erro "command not found". O que faço? ▼
Isso significa que o programa não foi encontrado. Soluções: 1) Feche e reabra o PowerShell após instalar qualquer coisa. 2) Verifique se instalou corretamente (node --version, git --version). 3) Se usou winget, reinicie o terminal. 4) Tente instalar pelo site como alternativa.
Posso usar AIOS sem o God Mode? ▼
Sim! O AIOS (npx aios-core install) instala o framework base com os agentes. O God Mode (npx create-aios-god-mode) adiciona 22 skills extras, 3 servidores MCP e 4 squads pré-prontos. Comece com o God Mode se quer tudo de uma vez, ou com o AIOS básico se quer ir devagar.
O Lovable vai "quebrar" se eu instalar AIOS pelo terminal? ▼
Não! O AIOS adiciona arquivos de configuração (na pasta .aios-core) que não interferem no código do seu app. O Lovable continua funcionando normalmente. Quando sincronizar de volta, o Lovable simplesmente ignora os arquivos que não reconhece.
O que é "npm install" e por que preciso rodar toda hora? ▼
npm install baixa todas as bibliotecas (dependências) que o projeto precisa. É como ir ao supermercado comprar os ingredientes de uma receita. Você roda quando: clona o projeto pela primeira vez, puxa atualizações do Lovable/GitHub, ou quando alguém adiciona uma biblioteca nova.
Posso ter Terminal e Desktop abertos ao mesmo tempo? ▼
Cuidado! Se ambos estiverem na mesma pasta, podem conflitar. O ideal é: use um OU outro por vez para o mesmo projeto. O Desktop com suas worktrees isola sessões automaticamente, mas se o Terminal também estiver editando, pode criar conflitos no Git.
O que fazer se der conflito no Git? ▼
Conflitos acontecem quando duas pessoas (ou o Lovable + você) editam o mesmo trecho de código. No Terminal, rode claude "resolva os conflitos de merge". No Desktop, peça ao Claude para resolver. Ele mostra as diferenças e propõe a solução. Depois faça git add . e git commit.
