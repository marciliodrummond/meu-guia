O que são Squads?
Pense em equipes completas de especialistas — só que de IA
▼
Analogia simples: Imagine que você quer construir uma casa. Você não contrata uma pessoa só — contrata um time: arquiteto, engenheiro, pedreiro, eletricista, encanador. Cada um sabe fazer uma coisa. Squads funcionam exatamente assim, mas com agentes de IA.

O AIOS já vem com 10 agentes padrão focados em desenvolvimento de software (@dev, @qa, @architect...). Mas e se você precisa de uma equipe de advogados? De médicos? De criadores de conteúdo? É para isso que existem os Squads.

Agentes Padrão vs. Squads
10 Agentes Padrão
Vêm com o AIOS. Focados em desenvolvimento de software.
+
Squads Personalizados
Você cria. Para qualquer área. Com quantos agentes quiser.
O mais incrível: você cria um squad em português, descrevendo o que quer. O AIOS monta toda a equipe — agentes, tarefas, hierarquia, fluxos — automaticamente.

Escolha: Squad Local ou Global?
Entenda a diferença antes de criar
▼
Squad de Projeto (local)
Fica só neste projeto. Ideal quando o squad é específico para uma aplicação. Salvo na pasta squads/ dentro do projeto.
Comando
/squad-creator-project
Squad Global (user)
Fica disponível em qualquer projeto que você abrir com Claude Code. Instalado globalmente na sua máquina.
Comando
/squad-creator-user
Dica prática: Use project para squads de nicho (ex: squad só para este e-commerce). Use user para squads que você quer reusar (ex: squad jurídico que serve para vários clientes).

4

Escolha como fornecer a documentação

O Squad Creator vai perguntar como você quer descrever seu squad. São 5 opções:

1. Colar texto/documento — Já tem um PRD, especificação ou briefing pronto? Cole aqui.
2. Caminhos de arquivo — Tem arquivos no computador? Indique o caminho.
3. Descrever verbalmente — A mais usada! Descreva em português o que quer.
4. Google Drive — Buscar documentação direto do seu Drive.
5. Outro — Método personalizado.
5

Descreva o que quer (a parte mais importante!)

Aqui vai a regra de ouro: quanto mais detalhes, melhor o resultado. O input é proporcional ao output.

O que incluir na descrição:
• Domínio/nicho — Sobre o que é? (ex: direito previdenciário, saúde mental, marketing digital)
• Objetivo — O que o squad deve fazer? (ex: criar petições, montar planos de marketing)
• Agentes sugeridos — Se já tem ideia de quem precisa na equipe (ex: advogado sênior, redator, revisor)
• Workflows — Quais os fluxos de trabalho? (ex: análise → redação → revisão → entrega)
• Integrações — Ferramentas externas? (ex: API do WhatsApp, planilhas)
• Público-alvo — Quem vai usar o resultado?
Macete: Não precisa preencher tudo. Escreva pelo menos o domínio, o objetivo e os agentes que imagina. O Squad Creator completa o resto e te pergunta se quer ajustar.
6

Revise e ajuste os agentes

O AIOS vai sugerir uma lista de agentes. Você pode:

Aceitar todos — se gostou da sugestão
Revisar — modificar nomes, funções ou instruções
Adicionar mais — digite "Adicione [nome do agente]"
Você pode adicionar agentes a qualquer momento escrevendo, por exemplo: Adicione um agente de storytelling e um de UX
7

Revise e ajuste as tasks (tarefas)

Para cada agente, o AIOS cria tarefas com input (o que recebe) e output (o que entrega). Revise se faz sentido e ajuste se necessário.

8

Gerar o Blueprint e criar!

Confirme tudo e o AIOS gera o blueprint final — um resumo com total de agentes, tasks, ajustes e uma nota de confiança (%). Depois ele cria toda a estrutura de arquivos.

A criação pode demorar alguns minutos. Isso é normal e é bom sinal — significa que o AIOS está criando uma estrutura robusta e detalhada. Quanto mais demora, melhor o resultado.
4
O que o Squad Creator gera por dentro
Entenda a estrutura de arquivos que é criada
▼

Quando o squad é criado, ele gera uma estrutura completa. Não é um simples texto — é uma arquitetura robusta:

squads/squad-nome/ ├── config.yaml # Configuração geral do squad ├── agents/ # Definições detalhadas de cada agente │ ├── agente-lider.md # Instruções, personalidade, skills │ ├── agente-2.md │ ├── agente-3.md │ └── ... ├── tasks/ # Tarefas de cada agente (input/output) │ ├── task-briefing.md │ ├── task-execucao.md │ └── ... ├── templates/ # Modelos reutilizáveis ├── checklists/ # Listas de verificação └── data/ # Dados de referência do domínio
Importante: Os arquivos do squad ficam na pasta squads/ e não interferem no código do seu app. Se você sincronizar de volta com o Lovable, ele simplesmente ignora esses arquivos.
5
Usando o Squad Criado
Como acionar os agentes e pedir coisas
▼

Depois que o squad foi criado, você usa os agentes como qualquer outro do AIOS — com @nome-do-agente:

Exemplos de uso
# Chamar o agente líder do squad (ex: creative-director)
@creative-director Crie um briefing para uma landing page de curso de IA

# Chamar um agente específico
@copywriter Escreva as copies para a seção hero e CTA

# Pedir que o squad inteiro trabalhe junto
Usando o squad web-design, crie uma landing page completa
para vender consultoria de IA para empresários.
Estilo: corporativo confiável. Seções: hero, benefícios,
depoimentos, FAQ, formulário de contato.
Como o Squad funciona por dentro:
O AIOS orquestra automaticamente. O agente líder recebe seu pedido, cria o briefing e depois vai acionando cada agente na ordem certa. Você pode acompanhar em tempo real quais tasks estão sendo executadas — quando ficam laranja/verde, foram concluídas.
Dicas importantes:
• O squad pode pedir confirmações durante a execução — aceite com y (yes) ou s (sim)
• Se precisa sair, pode voltar depois — o estado é salvo
• Para trabalho autônomo prolongado, use o modo yolo (aceita tudo sem perguntar)
• Para mudar de modelo (economizar tokens), use /model no Claude Code

Dicas e Macetes Avançados
Truques para squads de alto nível
▼

1. Squad que cria Squads

Você pode criar um squad cujo único objetivo é projetar outros squads melhores. Esse "meta-squad" analisa suas necessidades e sugere a melhor composição de equipe antes de criar o squad final.

Ideia de prompt
Quero um squad chamado "nirvana-creator" cujo objetivo é
analisar qualquer ideia de projeto e projetar o melhor
squad possível para executá-lo. Agentes: analista de
requisitos, arquiteto de squads, especialista em agentes,
e revisor de qualidade de squads.

2. Use o CLAUDE.md para regras globais

O arquivo CLAUDE.md (criado automaticamente pelo Claude Code) funciona como "leis" que todos os agentes seguem. Coloque ali regras como: "sempre escreva em português", "use acentuação correta", "siga o estilo da marca X".

3. Gerencie tokens com inteligência

Use /model para trocar de modelo. Para planejamento, Haiku é suficiente. Para execução complexa, use Opus. Para o meio-termo, Sonnet.

Trocar modelo
/model
# Selecione: haiku (barato) | sonnet (médio) | opus (melhor)

4. Arraste arquivos para o terminal

Você pode arrastar imagens, PDFs ou qualquer arquivo diretamente para a janela do Claude Code. O agente vai entender o conteúdo e usar como referência para o que está construindo.

5. Atualize squads sem perder customizações

Se precisar reinstalar o AIOS, seus squads na pasta squads/ são preservados. O comando de atualização é não-destrutivo.

Atualizar AIOS
npx aios-core@latest install

Vocabulário de Programação para Leigos

Clique em cada termo para ver a definição e analogia.

Fundamentos do Código
Código / Code"code"
▼
As instruções escritas que dizem ao computador o que fazer.
Se o programa é um prato pronto, o código é a receita.
Repositório (Repo)"repo"
▼
A pasta principal do projeto no GitHub. Guarda todo o código e histórico.
O armário de documentos do projeto — tudo organizado com registro de quem mexeu.
Frontend e Backend"front-end" e "bec-end"
▼
Frontend = o que o usuário vê. Backend = o motor invisível.
Frontend é o salão do restaurante. Backend é a cozinha.
API"ei-pi-ai"
▼
Canal de comunicação entre sistemas.
O garçom — leva seu pedido para a cozinha e traz o prato pronto.
Banco de Dados"deita-beis"
▼
Onde o programa guarda informações organizadas.
O fichário do escritório — cada gaveta com informações por categoria.
Versionamento (Git)
Commit"comit"
▼
Ponto de salvamento no código com mensagem.
Ctrl+S com superpoderes — salva E cria registro na linha do tempo.
Branch"brench"
▼
Cópia paralela do código para trabalhar sem bagunçar a versão principal.
Rascunho separado do documento oficial.
Pull Request (PR)"pul ri-quest"
▼
Pedido formal para juntar seu código à branch principal.
Entregar relatório para o chefe revisar antes de publicar.
Push e Pull"puch" e "pul"
▼
Push = enviar para o GitHub. Pull = baixar do GitHub.
Push = upload no Drive. Pull = download do Drive.
Execução e Entrega

Perguntas Frequentes

Preciso saber programar para usar tudo isso? ▼
Não! O Claude Code e os agentes AIOS entendem português natural. Basta descrever o que quer.
Qual a diferença entre Terminal e Claude Desktop? ▼
Terminal: Tela preta, mais controle. Desktop: App visual, vê diffs. O motor é o mesmo — os comandos funcionam igual.
Preciso pagar? Quanto custa? ▼
Conta paga do Claude: Pro (~US$20/mês) ou Max (~US$100/mês). NÃO precisa de API Key se tem Pro/Max.
Funciona no Mac/Linux? ▼
Sim! No Mac use o Terminal. No Linux, instale via gerenciador de pacotes. Comandos quase idênticos.
Preciso instalar o AIOS separadamente do God Mode? ▼
Não! O God Mode já instala o AIOS automaticamente.
O Lovable vai "quebrar" se eu instalar AIOS pelo terminal? ▼
Não! O AIOS adiciona arquivos em .aios-core que não interferem no app. O Lovable ignora esses arquivos.
O que é um Squad e qual a diferença dos agentes padrão? ▼
Os 10 agentes padrão (@dev, @qa, etc.) são focados em desenvolvimento de software. Squads são equipes customizadas que você cria para qualquer área: jurídico, saúde, marketing, educação, criação de livros... As possibilidades são ilimitadas.
Qual a diferença entre squad-creator-project e squad-creator-user? ▼
Project: O squad fica só naquele projeto. User: O squad fica global na sua máquina e aparece em qualquer projeto que você abrir.
Posso criar squads para áreas que não são de tecnologia? ▼
Sim! Squads funcionam para qualquer domínio: jurídico, saúde, finanças, educação, marketing, criação de livros, administração de sistemas, imobiliário... O AIOS trabalha com linguagem natural em qualquer área.
Quanto tempo leva para criar um squad? ▼
O processo interativo (perguntas + ajustes) leva de 5 a 15 minutos. A geração dos arquivos pode levar mais alguns minutos. Squads mais complexos demoram mais, mas o resultado é proporcional — quanto mais tempo, mais robusto.
O squad consegue trabalhar sozinho enquanto eu durmo? ▼
Em grande parte, sim. Use o modo yolo para ele aceitar confirmações automaticamente. Porém, em ações críticas (como deletar arquivos), ele pode pedir confirmação manual.
Por que o resultado veio sem acentuação? ▼
Isso pode acontecer dependendo do modelo e configurações. A solução é adicionar no arquivo CLAUDE.md a regra: "Sempre escreva em português do Brasil com acentuação correta". Todos os agentes e squads seguem as regras desse arquivo.
Posso usar squads com outras IDEs além do Claude Code? ▼
Sim. Na instalação do AIOS, você pode escolher entre Claude Code, Cursor, Windsurf, Gemini CLI e outros. Os squads funcionam da mesma forma em qualquer IDE compatível.
Como economizar tokens ao usar squads? ▼
Use /model para trocar de modelo. Para planejamento e briefing, use Haiku (mais barato). Para execução complexa, use Opus (melhor qualidade). Lembre-se: o custo é proporcional à qualidade.
